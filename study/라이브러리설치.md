라이브러리 - 개발 시 자주 사용하는 코드를 모아 둔 것

## Spring Boot Devtools 설치

* 출력하는 문자열을 "Hello Word"에서 'Hello SBB'로 변경하더라도 로컬호스트/hello URL을 호출해도 수정하기 전이었던 Hello Word가 출력된다.
* 프로그램이 변경되더라도 별도의 과정 없이는 로컬 서버가 변경된 클래스를 즉시 반영하지 않기 때문이다.
* 변경된 사항을 확인하기 위해 매번 서버를 재시작해야 하므로 개발 과정이 번거롭다.

* 이러한 문제를 해결하기 위해 Spring Boot Devtools를 설치한다.
* Spring Boot Devtools는 서버를 재시작하지 않아도 클래스를 변경할 때 서버가 자동으로 재가동됨.

## 롬복 설치

* lombok 라이브러리는 소스 코드를 작성할 때 자바 클래스에 애너테이션을 사용하여 자주 쓰는 Getter 메서드, Setter 메서드, 생성자 등을 자동으로 만들어 주는 도구이다.
* 조금 더 짧고 깔끔한 소스 코드를 만들 수 있다는 장점이 있음
* SBB 프로그램을 만들면서 게시물과 관련된 데이터를 처리하기 위해 엔티티 클래스나 DTO 클래스 등을 사용해야 하는데 그러기 위해서는 먼저 이 클래스들의 속성값을 읽고 저장하는 Getter, setter 메서드를 만들어야 한다.

롬복 플러그인 설치(https://projectlombok.org/download)

<img width="642" height="423" alt="image" src="https://github.com/user-attachments/assets/bb51c5b7-3a90-4d17-96f4-54f08422a91e" />

Getter,Setter 애너테이션을 적용했더니 Getter,Setter 메서드를 다로 작성하지 않아도 사용할 수 있게 되었다.

* 컴파일 단계에서 롬복이 대신 생성해 주므로 컴파일된 클래스에는 Getter, Setter 메서드가 포함된다.

<img width="827" height="396" alt="image" src="https://github.com/user-attachments/assets/3e2dae01-f76e-4257-8eb6-76f8d5db10ba" />

속성에 final을 추가하고 @RequiredArgsConstructor 애너테이션을 적용하면 해당 속성을 필요로 하는 생성자가 롬복에 의해 자동으로 생성된다.

* final은 자료형과 변수 등을 변경할 수 없게 만드는 키워드, final이 없다면 생성자에 포함되지 않는다.
* final을 적용하면 속성값을 변경할 수 없기 때문에 @Setter는 의미가 없어진다.
